# The Living Atlas Documentation

This directory contains the complete documentation for The Living Atlas Knowledge Platform.

## Documentation Structure

The documentation is organized in a layered architecture following the principle of separation of concerns:

- **product/** - Product vision, principles, and strategy
- **experience/** - User experience specifications (Experience PRDs)
- **frontend/** - Frontend platform architecture and specifications
- **design-system/** - Design system, tokens, and component library
- **backend/** - Backend platform architecture and specifications
- **ai-platform/** - AI platform architecture and specifications
- **database/** - Database schemas and strategies
- **architecture/** - System architecture and design patterns
- **workflows/** - Business process workflows
- **api/** - API guidelines and specifications
- **security/** - Security architecture and guidelines
- **deployment/** - Deployment and infrastructure
- **analytics/** - Analytics and metrics
- **adr/** - Architecture Decision Records
- **glossary/** - Domain and technical glossaries

## Quick Start

Start with the [Product Overview](product/00-product-overview.md) to understand the platform vision.

## Documentation Philosophy

This documentation follows a layered architecture approach:

1. **Product** defines *why* the platform exists
2. **Experience** defines *how users interact* with the platform
3. **Frontend** defines *how experiences are implemented*
4. **Design System** provides consistent visual language
5. **Backend** defines the technical implementation
6. **AI Platform** defines AI/ML capabilities
7. **Database** defines data storage and management
8. **Architecture** defines system-wide patterns and decisions

This structure ensures documentation remains maintainable as the platform grows.
